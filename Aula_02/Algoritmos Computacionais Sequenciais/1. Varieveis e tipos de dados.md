# Variáveis

 Toda informação contida dentro de um programa está armazenada na memória do computador. Nossos dados, em particular, estão sempre guardados dentro de uma variável. O conceito de variável varia um pouco a depender da área. Na matemática, uma variável é uma entidade capaz de representar um valor ou expressão. Ela pode representar um número ou um conjunto de números, como na equação 
```
 x² + 2x + 1 = 0
```
ou na função
```
 f(x) = x²
```

  Já em computação, uma variável é uma posição de memória onde poderemos guardar determinado dado ou valor e modificá-lo ao longo da execução do programa. Quando criamos uma variável e armazenamos um valor dentro dela(por exemplo x = 25;), o computador reserva um espaço associado a um endereço de memória onde podemos guardar o valor dessa variável.

  Como no Exemplo a seguir;



   ## Memória

| Endereço | Variável | Conteúdo |
|----------|----------|----------|
| 119      |          |          |
| 120      |          |          |
| 121      | int **X**|    10    |
| 122      |          |          |
| 123      |          |          |


  Desse modo, podemos recuperar esse valor ao longo do programa e utilizá-lo nas mais diversas operações.

  Então desse modo, uma variável acaba sendo uma abstração na memória do nosso computador, o que nós chamamos de "endereço de memória" e é bem melhor que os programadores se refiram a uma posição de memória por um nome da sua preferência do que por endereços diretos de memória.

  Um programa onde nós temos que se referir diretamente à posição de memória fica um tanto quanto estranho. Imagine se tivessemos que nos referir assim à um endereço de memória;

#### O CÓDIGO-FONTE ABAIXO É INVÁLIDO!!

```
#include<stdio.h>

int main(){

    0x7fff9779d954 = 2;

    return 0;
}
```

  Podemos notar que é uma complexidade desnecessária. Isso já aconteceu em computadores mais antigos onde nós tinhamos que programar nos referindo a cada endereço de memória específico (veja [aqui sobre os computadores com arquitetura de memória explícita](https://pt.wikipedia.org/wiki/IBM_System/360)). Mas hoje em dia isso se torna impensável considerando as linguagens de alto nível (a Linguagem C), em que temos programas mais complexos do que antigamente. Atualmente, os compiladores em conjunto com o sistema operacional assumem a responsabilidade de alocar o espaço de memória para as variáveis declaradas no programa.

# Como nomear uma variável?

### Declarando uma variável

  Em linguagem C, a declaração de uma variável pelo programador segue a esta forma geral:

```
  TIPO_DA_VARIÁVEL nome_da_variável;
```

  O **tipo_da_variável** determina o conjunto de valores e de operações que uma variável aceita, ou seja, que ela pode executar. Já o **nome_da_variável** é como o programador identifica essa variável dentro do programa. Ao nome da variável o computador associa o endereço do espaço que ele reservou na memória para guardá-la.
  Suponha que queremos declarar uma variável do tipo inteiro e de nome **x**. Ela pode facilmente declarada como apresentado a seguir:

```
int X
```

 Além disso, mais de uma variável pode ser declarada ao mesmo tempo. para tanto, basta separar cada nome de variável por uma **Vírgula** (,):

```
int x,y,z;
```

**TODA DECLARAÇÃO DE VARIÁVEIS TERMINA COM O OPERADOR DE PONTO E VÍRGULA(;)**

  Isso é necessário uma vez que o operador de **ponto e vírgula** é utilizado para separar as instruções da linguagem C que compõem um programa de computador.

**UMA VARIÁVEL DEVE SER SEMPRE DECLARADA ANTES DE SER USADA NO PROGRAMA**

  Lembre-se: apenas quando declaramos uma variável é que o computador reserva um espaço de memória para guardarmos nossos dados. Assim, não é possível usar esse espaço antes de tê-lo reservado na memória.
  Quando falamos em memória do computador, não existe o conceito de posição de memória **"vazia"**. Toda posição de memória do computador está preenchida com um conjunto de 0s e 1s. O que ocorre é que a posição de memória pode apenas não estar sendo utilizada. Portanto, ao criarmos uma variável e, portanto, ela automaticamente estará preenchida com um valor chaamado de **"lixo de memória"**: um conjunto de 0s e 1s sem significado para nosso programa.

### Nomeando uma variável

  Quando criamos uma variável, o computador reserva um espaço de memória onde podemos guardar o valor associado a ela. Ao nome que damos a essa variável o computador associa o endereço do espaço que ele reservou na memória para guardá-la. De modo geral, interessa ao programador saber o nome das variáveis na Linguagem C.
  Os nomes das variáveis podem ter vários caracteres, no qual apenas os 31 primeiros são considerados. As regras são as seguintes:

- O primeiro caractere tem que ser uma letra ou subliminhado "_;"
- O restante do nome da variável pode conter letras, dígitos e sublinhados;
- Palavras reservadas da Linguagem C não podem ser utilizadas como variáveis, como ```int```,```while```,```return```,```main``` etc.;
- existe distinção entre letras maísculas e minúsculas na Linguagem C. Logo as variáveis com os nomes "flag", "Flag" e "FLAG", são diferentes;
- **Dica:** Coloque nome para as variáveis de acordo com seu papel ou funcionamento dentro do código-fonte.
- O nome de uma variável é um conjunto de caracteres que podem ser letras, números ou o underscore (_).

  A Linguagem C é **case-sensitive**, ou seja, uma palavra escrita utilizando caracteres maiúsculos é diferente da mesma palavra escrita com **caracteres minúsculos.** Sendo assim, o exemplo da palavra **"flag"** acima. Essas variáveis são distintas.

### Palavras-Chave não podem ser usadas como nome de variáveis.

  As palavras-chave são um conjunto de 32 palavras reservadas dentro da linguagem C. São elas que formam a sintaxe da linguagem de programação e, por esse motivo, não podem ser utilizadas para outro fim, como para nome de variáveis.

  | tipo | Bits | Intervalo de valores |
  |------|------|----------------------|
  | char |  8 |      -128 A 128      |
  | int  | 32 | -2.147.483.648 A 2.147.483.647 |
  | float | 32 |  1,175494E-038 A 3,402823E+038 |
  | double | 64 |  2,225074E-308 A 1,797693E+308 |
  | void |  8 | sem valor |

 ## char

   Começaremos pelo o tipo **char**. Esse tipo de dados permite em um único byte (8 bits) um número inteiro ou o código de caractere, conforme a [Tabela ASCII](https://www.ime.usp.br/~kellyrb/mac2166_2015/tabela_ascii.html). Vale ressaltar que quando armazenamos um número inteiro, aquele número representará um caractere específico da Tabela ASCII. É bem comum usarmos os caracteres da própria Tabela.

   O código-fonte abaixo demonstra como você pode armazenar um dado na variável do tipo ```char```, Com o nome ```var_char```, é declarada e um valor constante lhe é atribuído. Na nossa variável ```var_char_int```, lhe é atribuído um número inteiro, onde na tabela ASCII o código 43 representa "+".

```
    #include <stdio.h>

    int main(){
        char var_char,var_char_int;

        var_char = 'e'

        var_char_int = 43; //vai armazenar o caractere '+'
    
        return 0;
    }
```
   ### **Caracteres sempre ficam entre aspas simples!!**

   Lembre-se: uma única letra pode ser o nome de uma variável. As **aspas simples** permitem que o compilador saiba que estamos iniciando nossa variável com um caractere e não com o conteúdo de outra variável.


## int

   O segundo tipo de dado é o tipo inteiro: **int**. Esse tipo de dados permite armazenar um número inteiro (sem parte fracionária). Permite representar um valor inteiro (negativo, zero ou positivo) em uma variável que ocupa 4 bytes na memória. Com isso, essa variável armazena exatamente valores entre o intervalo [-32.768,32.767O], o que é aproximadamente [-2 x 10^-9,2 x 10^9.] O código-fonte seguinte mostra a declaração de uma variável do tipo int, chamada var_int, e o valor constante 16 lhe é atribuído:

```
    #include <stdio.h>

    int main(){
        int var_int;

        var_int = 16;

        return 0;
    }
```

   Cuidado com a forma como você inicia as variáveis dos tipos **char** e **int.**

   Na Linguagem C, os tipos **char** e **int** podem ser especificados nas bases **decimal**, **octal** e **hexadecimal**. A base **decimal** é a padrão. Porém, se o valor inteiro for precedido por: 

  - "0", ele será interpretado como hexadecimal. Nesse caso, o valor deve ser definido utilizando digitos de 0, 1, 2, 3, 4, 5, 6 e 7. EX.: **int** x = 044. Nesse caso, 044 equivale a 36(4 * 8¹ + 4 * 8^0).
  - "0X" ou "0x", ele será interpretado como hexadecimal. Nesse caso, o valor deve ser definido utilizando digitos utilizando digitos de 0, 1, 2, 3, 4, 5, 6, 7, 8 e 9, e as letras A(10), B(11), C(12), D(13), E(14) e F(15). EX.: **int** = 0x44. Nesse caso, 0x44 será equivalente a 68(4 * 16¹ + 4 * 16^0).

   Segue o exemplo abaixo, **rode o seguinte código-fonte em seu computador**;

```

    #include <stdio.h>

    int main(){
        int a,b,c;

        a = 125;
        b = 0435; //valor em octal, equivale a 285
        c = 0x1FA; //valor em hexadecimal, equivale a 506

        printf("valor de a: %d\nvalor de b: %d\nvalor de c: %d\n",a,b,c);

        return 0;
    }     

  ```

   **ATENÇÃO!!** Cuidado com o **OVERFLOW** que podem ocorrer em variáveis do tipo ```int``` após operações de leitura de dados ou aritméticas. Veja o exemplo abaixo:

   ```
    #include <stdio.h>

    int main(){
        int a,b,p;

        a = 1000000000;
        b = 1000000000;

        p = a*b; /* OPA!! OVERFLOW!!*/

        printf("Exemplo de Overflow: %d\n", p);
        
        return 0;  
    }

 ```

   **Rode o código em seu computador e veja o que é Overflow.**

   O que o código-fonte acima faz?? O que ele imprime na tela?? Podemos perceber que a capacidade de memória no meu programa é extrapolada e passa a quantidade de bytes que o tipo ```int```suporta. Em modificadores você descobre como resolver o problema. Já que nós precisamos de um tipo de dados que armazene mais informações...

## float

   O terceiro tipo é o tipo float, o float permite o programador armazenar números reais com até 7 dígitos de precisão. Ou seja, considerando todos os dígitos da parte inteira e fracionária(real, de fato) desse número, conhecido como **ponto flutuante**. Uma variável do tipo ```float``` tem capacidade de armazenamento de 4 bytes.

   Definição: A precisão se refere aos dígitos que informam o valor do número, excluindo zeros à esquerda e zeros à direita após a parte decimal. Por Exemplo, o número ```72.98776``` tem sete dígitos de precisão, em que 2 deles estão na parte inteira e 5 na parte fracionária. Quanto mais dígitos de precisão um número tem, mais detalhes podem ser efetivamente representados.

   No código-fonte abaixo, declaramos a variável ```var_f``` como sendo do tipo ```float``` e atribuímos o valor 72.98776 a ela.

```
    #include <stdio.h>

    int main(){
        float var_f;

        var_f = 72.98776;

        return 0;
    }  
```

## double

   O quarto tipo é o tipo o double, o double nos permite armazenar números reais com até 15 dígitos de precisão, o que é aproximadamente o dobro de um valor real no tipo ```float```. Isso se deve ao fato da variável possuir capacidade de armazenamento de 8 bytes.

   No código-fonte abaixo, a variável ```var_db``` é declarada como sendo do tipo ```double``` e recebe um valor real com muitos dígitos, mas que são o suficiente para serem armazenados:

```
    #include <stdio.h>

    int main(){
        double var_db;

        var_db = 123561.0238495334;

        return 0;
    }
```

## Os tipos float e double

   O terceiro e o quarto tipo de dados são os tipos reais: **float** e **double**. Ambós são conhecidos como **ponto flutuante**. a diferença entre eles é a sua **precisão!!**

   - Tipo **float:** precisão simples.
   - Tipo **double:** dupla precisão.

   São úteis quando queremos trabalhar com intervalos de números reais realmente grandes.

   ### **Em números reais, a parte decimal usa ponto, e não vírgula!**

   A Linguagem C usa o padrão número Estadunidense, ou seja, a parte decimal fica depois de um ponto. Veja os exemplos:

```
    #include <stdio.h>

    int main(){
        float = 5.25;
        double d = 15.673

        return 0;
    }
```

   ### Pode-se escrever números dos tipos **float** e **double** usando notação científica.

   A notação científica é uma forma de escrever números extremamente grandes ou extremamente pequenos. Nesse caso, o valor real é seguido por uma letra "e" ou "E" e um número inteiro (positivo ou negativo), que indica o exponte da base 10 (representado pela letra "e" ou "E" que multiplica o número):

```
    double x = 5.0e10;
```

   ### Equivale a
   
```
    double x = 50000000000;
```

### Observação importante

   Evite o uso de variáveis do tipo **float** devido a sua baixa precisão que podem ocasionar cálculois imprecisos na parte fracionária. Ademais, nem todo valor real pode ser representado por variáveis **float**.

### void
   
   Por fim, temos o tipo **void**. Esse tipo de dados nos permite declarar uma função que não retorna valor ou um ponteiro genérico, como será visto mais pra frente.

### A linguagem C não permite que se declare uma variável do tipo **void**. Esse tipo de dados só deve ser usado para declarar funções que não retornam valor ou ponteiro genérico.

## Modificadores

  Os modificadores de tipos são aplicados precedendo os tipos básicos (com a exceção do tipo **void**), e permitem alterar o significado do tipo, de modo a adequá-lo as necessidades do nosso programa. São basicamente utilizados para variações dos tipos de dados existentes na linguagem C. Os modificadores são:

  - Signed
  - Unsigned
  - short
  - long

### signed

  Esse modificador determina que uma variável declarada dos tipos **char** ou **int** poderá ter valores positivos ou negativos. Trata-se do **modo-padrão** de definição de variáveis desses tipos, e, por esse motivo raramente é usado.

  ```
#include <stdio.h>

int main(){
    signed char x;
    signed int y;

    return 0;
}
```
### Unsigned

   Esse modificador determina que uma variável declarada dos tipos **char** ou **int** somente poderá ter valores positivos e o valor zero. Nesse caso, a variável perde seu bit de sinal, o que dobra sua capacidade de armazenamento para valores positivos. Por exemplo, uma variável do tipo **char** é capaz de armazenar valores de -128 até 127. Se a mesma variável for declarada como sendo do tipo **unsigned char**, ela será capaz de armazenar valores de 0 até 255. O código abaixo mostra a declaração de duas variáveis inteiras sem sinal para os tipos **int** e **long**:

```
#include <stdio.h>

int main(){
    unsigned int a;
    unsigned long long int b;

    a = 5;
    b = a+1;

    return 0;
}
```

### short

   O modificador **short** determina que uma variável do tipo **int** terá apenas 16 bits (*inteiro pequeno*), independentemente do processador.

### long ou long long

   Faz o inverso do modificador **short**. O modificador **long** determina que uma variável do tipo **int** terá 32 bits (*inteiro grande*), idependentemente do processador. Também determina que o tipo **double** possua maior precisão.

### é possível utilizar mais de um modificador ao mesmo tempo??

   **A linguagem C permite que se utilize mais de um modificador de tipo sobre um mesmo tipo.**

   Isso permite, por exemplo, declarar um inteiro grande(ou seja, com 32 bits) usando o modificador (**long**) e que também seja sem sinal (**unsigned**). Essa combinação permite aumentar em muito o intervalo de valores possíveis para aquela variável:

```
#include <stdio.h>

int main(){
    unsigned long int m;

    return 0;
}
```
